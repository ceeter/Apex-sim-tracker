Forza Horizon 6 — Data Out

FH6 (Japan, May 2026) sends a fixed 324-byte UDP packet per frame — identical format to FH5. The bridge script receives it and streams to this app over WebSocket.

In-Game Setup

1
Settings → HUD & Gameplay → Data Out → ON
2
Set IP Address to your PC's local IP (ipconfig in Command Prompt)
3
Set Port to 20066
4
No format selection — FH6 has one fixed format

Bridge Script

1
Run node apex-bridge.js (included alongside this file)
2
Select FH6 mode when prompted, or pass --game fh6
3
Connect below at ws://localhost:8765

F1 25 — UDP Telemetry

F1 25 sends structured UDP packets on port 20777. Use format 2025. The bridge parses lap times, car status, and motion data.

In-Game Setup

1
Main Menu → Settings → Telemetry Settings
2
Set UDP Telemetry to On
3
Set UDP IP Address to your PC's local IP
4
Set UDP Port to 20777
5
Set UDP Format to 2025
6
Set UDP Send Rate to 20Hz (recommended)

Bridge Script

1
Run node apex-bridge.js --game f1
2
Bridge listens on UDP 20777, streams to WebSocket 8765

iRacing — SDK / Memory Map

iRacing doesn't use UDP. It writes live telemetry to a Windows shared memory map at 60Hz. The bridge reads it via the iRacing SDK and forwards to this app over WebSocket.

⚠️ Windows only. Requires iRacing to be running (you don't need to be in a session — the sim service must be active).

Bridge Setup

1
Install Node.js v18+ from nodejs.org
2
In the same folder as the bridge: npm install irsdk-node ws
3
Start iRacing (the iRacingSVC service must be running)
4
Run node apex-bridge.js --game ir
5
Bridge auto-connects when iRacing SDK is detected
